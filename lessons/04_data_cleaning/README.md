# Lesson 04 — Data Cleaning

## Learning Goals

By the end of this lesson, students will be able to:

1. Enumerate at least four categories of dirty data and give a concrete example of each from a reaction-time experiment.
2. Apply standard RT exclusion criteria (anticipations < 100 ms, lapses > 2000–3000 ms, timeouts) and calculate the exclusion rate for a real dataset.
3. Explain why manual CSV editing is not an acceptable cleaning method and describe what reproducible cleaning looks like in practice.
4. Describe the role of pre-registration and cleaning logs in preventing p-hacking.

## Interactive Activity

No interactive activity is available for this lesson yet. A data-cleaning simulator is planned for a future sprint.
